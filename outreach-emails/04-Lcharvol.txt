Hi Lucas,

I found MacSift recently and really like the approach — grouping files by app and using Trash-first deletion is exactly the right safety posture for a disk cleaner. The SwiftUI + Liquid Glass UI looks great too.

I'm building X-MaC, which tackles a similar problem from a slightly different angle. It has a Rust core for scanning (disk usage, cache bloat, Xcode artifacts, PATH conflicts, broken symlinks, missing dylibs, environment mapping) and a SwiftUI frontend. It also generates safe remediation scripts and includes a GNN model (CoreML) for filesystem anomaly detection.

Since we're both building open source macOS cleaners, I think there's a lot we could learn from each other. I'd love to hear your thoughts:

https://github.com/davidnichols-ops/X-MaC

Discussions are enabled. Whether it's feedback, collaboration, or just swapping notes on the challenges of building macOS system tools — I'm up for it.
